# 구현 사항 (Implementation)

## 1. 공통 모듈 및 라이브러리화
- `common.py`: 모든 입력 처리 및 유틸리티 로직 통합.
- `icon.py`, `logo.py`: CLI 의존성을 제거하고 순수 생성 함수(`generate_*`)만 제공하는 라이브러리 형태로 전환.
- **배경 제어 통합**: 아이콘과 로고 모두에서 `include_bg` 옵션을 지원하도록 리팩토링.

## 2. 통합 CLI 진입점 (`cli.py`)
- **단일 명령어 `uzbar`**: `argparse`를 사용하여 모든 기능을 하나의 명령어로 통합.
- **멀티 모드 지원**: `--icon`, `--logo`, `--all` 플래그를 통해 개별 또는 전체 자산 생성을 지원.
- **범용 옵션**: `--in`, `--out`, `--bg`, `--no-bg`, `--no-img` 등의 옵션이 모든 모드에 일관되게 적용됨.
- **텍스트 전용 모드 (`--no-img`)**: 브랜딩 시 아이콘 없이 텍스트 워드마크만 필요한 경우를 위해 아이콘 제외 렌더링 기능 구현.
- **자동 경로 관리**: `--all` 모드나 디렉토리 입력 시 서비스별 하위 폴더를 자동으로 생성 및 관리.

## 3. 문서화 및 인프라
- `README.md`: 통합 CLI 사용법과 프로젝트 구조를 상세히 기술.
- `pyproject.toml`: 진입점을 `cli:main`으로 단일화하여 유지보수성 향상.
