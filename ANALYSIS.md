# 분석 (Analysis)

## 1. 기존 현황 분석
- `icon.py`, `logo.py`: 각각 독립적인 CLI 로직(`main`)을 가지고 있음.
- `uzbar`, `uzlogo` 등 개별 진입점이 분산되어 관리 및 사용이 불편함.

## 2. 신규 요구사항 분석 (통합 CLI)
- **단일 진입점**: `uzbar` 명령어로 모든 기능을 통합.
- **명령어 체계**:
  - `uzbar --icon`: 아이콘 생성 (기본 512x512, 3개 레이블 위치).
  - `uzbar --logo`: 로고 생성 (가로형 워드마크).
  - `uzbar --all`: 아이콘, 로고 등 모든 자산 생성 (향후 확장 고려).
- **공통 옵션 유지**: `--in`, `--out`, `--bg`, `--no-bg` 등 기존 옵션을 범용적으로 적용.
- **범용적 구현**: 로직이 특정 타입에 종속되지 않도록 CLI 파싱과 핵심 생성 로직을 분리.

## 3. 구조 설계
- `cli.py`: 전역 `argparse` 설정 및 제어 흐름 담당.
- `icon.py`, `logo.py`: 순수 생성 라이브러리 역할로 변경 (CLI 로직 제거).
- `common.py`: 설정 로드 및 공통 유틸리티 유지.

## 4. 상세 동작 설계
- `uzbar --all` 실행 시:
  - 지정된 `--in` (또는 기본 `./input`) 환경에서 모든 데이터 로드.
  - 아이콘과 로고 경로를 구분하여 생성 (예: `output/{name}/icon/`, `output/{name}/logo/`).
- 배경 옵션: `--bg`/--no-bg`가 아이콘과 로고 모두에 적용되어야 함.
- **이미지 제외 옵션 (`--no-img`)**:
  - 아이콘(Lucide)이나 이모지를 렌더링하지 않고 워드마크(텍스트)만 포함된 SVG 생성.
  - 브랜딩 도구로서 텍스트만 필요한 경우를 대응.
- 아이콘 레이블 위치 옵션: `--icon`이 선택될 때만 유효하거나, 전체 생성 시 기본값(3종) 적용.
