# 분석 (Analysis)

사용자는 기존의 `icon.py` 스크립트에 `--all` 옵션을 통합하여 일괄 작업을 자동화하고자 합니다.

요구사항:
1. `--all` 옵션 추가.
2. 동작 방식:
   - `./input` 디렉토리 내의 모든 JSON 파일을 읽음.
   - 각 서비스(`name`)별로 별도의 하위 디렉토리를 생성: `output/{name}/`.
   - 각 디렉토리 내에 3가지 레이블 위치(`center`, `bottom`, `below`)에 대한 아이콘 파일 생성.
   - 파일명 형식: `icon-{label_pos}.svg`.

분석 결과:
- `--all` 플래그가 설정되면 `--in`과 `--out`의 기본값이 각각 `./input`과 `./output`으로 간주되어야 합리적입니다. (사용자 요청에 명시됨)
- `generate_icons` 함수 또는 `main` 로직에서 서비스별 디렉토리 생성 로직을 추가해야 합니다.
- 파일명 생성 규칙을 기존 `uz{name}-{label_pos}.svg`에서 `icon-{label_pos}.svg`로 변경해야 합니다. 이는 디렉토리가 서비스 이름으로 구분되므로 파일명에 서비스 이름을 명시할 필요가 없어지기 때문입니다.
- `--all` 모드가 아닐 때의 일관성도 고려해야 하지만, 사용자 요청 "파일명은 icon-{label_pos}.svg 여야 해"를 우선 적용합니다.

타입 불일치 에러 (해결됨):
- `service["icon"]` 직접 참조를 통해 정적 분석 에러를 해결함.
